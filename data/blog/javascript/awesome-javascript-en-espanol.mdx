## Gestores de paquetes

Alojan las bibliotecas de JavaScript y proporcionan herramientas para obtenerlas y empaquetarlas.

- [npm](https://www.npmjs.com/) - npm es el gestor de paquetes de JavaScript.
- [Bower](https://github.com/bower/bower) - Un gestor de paquetes para la web.
- [component](https://github.com/componentjs/component) - Gestión de paquetes de clientes para crear mejores aplicaciones web.
- [spm](https://github.com/spmjs/spm) - Nuevo gestor de paquetes estáticos.
- [jam](https://github.com/caolan/jam) - Un administrador de paquetes que usa un repositorio centrado en el navegador y compatible con RequireJS.
- [jspm](https://github.com/jspm/jspm-cli) - Gestión de paquetes de navegador sin fricciones.
- [Ender](https://github.com/ender-js/Ender) - La biblioteca sin bibliotecas.
- [volo](https://github.com/volojs/volo) - Crea proyectos front-end a partir de plantillas, agrega dependencias y automatiza los proyectos resultantes.
- [Duo](https://github.com/duojs/duo) - Administrador de paquetes de próxima generación que combina las mejores ideas de Component, Browserify y Go para que la organización y escritura de código front-end sea rápida y sencilla.
- [yarn](https://yarnpkg.com/) - Gestión de dependencias rápida, fiable y segura.
- [pnpm](https://pnpm.io/) - Administrador de paquetes rápido y eficiente en espacio de disco.

## Gestores de componentes

- [Bit](https://github.com/teambit/bit) - Crea, encuentra y reutiliza componentes (React, Angular, Node, etc.) en todas las aplicaciones.

## Loaders

Módulo o sistema de carga para JavaScript.

- [RequireJS](https://github.com/requirejs/requirejs) - Un cargador de archivos y módulos para JavaScript.
- [browserify](https://github.com/substack/node-browserify) - require() en el lado del navegador al estilo de node.js.
- [SeaJS](https://github.com/seajs/seajs) - Un cargador de módulos para la web.
- [HeadJS](https://github.com/headjs/headjs) - El único script en tu HEAD.
- [lazyload](https://github.com/rgrove/lazyload/) - Pequeño cargador asíncrono de JavaScript y CSS sin dependencias.
- [script.js](https://github.com/ded/script.js) - Cargador asíncrono de JavaScript y administrador de dependencias.
- [systemjs](https://github.com/systemjs/systemjs) - Cargador de módulos que cumple con las especificaciones de AMD, CJS y ES6.
- [LodJS](https://github.com/yanhaijing/lodjs) - Cargador de módulos basado en AMD.
- [ESL](https://github.com/ecomfe/esl) - ESL es un cargador de módulos compatible con AMD solo para navegador.
- [modulejs](https://github.com/lrsjng/modulejs) - Sistema de módulo de JavaScript ligero.

## Bundlers

- [webpack](https://github.com/webpack/webpack) - Empaqueta módulos CommonJs/AMD para el navegador.
- [Rollup](https://github.com/rollup/rollup) - Paquete de módulos ES6 de última generación.
- [Brunch](https://github.com/brunch/brunch) - Herramienta de creación de aplicaciones web front-end rápida con configuración declarativa simple.
- [Parcel](https://github.com/parcel-bundler/parcel) - Paquete de aplicaciones web increíblemente rápido y sin configuración.
- [Microbundle](https://github.com/developit/microbundle) - Paquete de configuración cero para módulos diminutos.
- [FuseBox](https://github.com/fuse-box/fuse-box) - Un empaquetador que lo hace bien
- [Snowpack](https://www.snowpack.dev/) - Una herramienta de creación de interfaz ultrarrápida, diseñada para la web moderna.

## Type Checkers

- [TypeScript](https://www.typescriptlang.org/) - Un superconjunto tipado de JavaScript que se compila en JavaScript simple.
- [Flow.js](https://flow.org/en/) - Un verificador de tipos estáticos para JavaScript de Facebook.
- [Hegel](https://hegel.js.org/) - Un verificador de tipos estáticos para JavaScript con un sesgo en la inferencia de tipos y un sistema de tipos fuerte.
- [TypL](https://github.com/getify/TypL) - el JavaScript Type Linter con un sesgo en la inferencia de tipos.
- [Hindley Milner Definitions](https://github.com/xodio/hm-def) - Comprobación de tipo en tiempo de ejecución para funciones de JavaScript utilizando firmas de tipo Hindley Milner similares a Haskell.

## Testing Frameworks

### Frameworks

- [mocha](https://github.com/mochajs/mocha) - Test framework de JavaScript simple, flexible y divertido para node.js y el navegador.
- [jasmine](https://github.com/jasmine/jasmine) - Testing framework de JavaScript simple sin DOM.
- [qunit](https://github.com/jquery/qunit) - Un framework de testing unitario de JavaScript fácil de usar.
- [jest](https://github.com/facebook/jest) - Pruebas unitarias de JavaScript sin dolor.
- [prova](https://github.com/azer/prova) - Node & Browser test runner based on Tape and Browserify.
- [DalekJS](https://github.com/dalekjs/dalek) - Pruebas funcionales automatizadas entre navegadores con JavaScript.
- [Protractor](https://github.com/angular/protractor) - Protractor es un framework de testing de extremo a extremo para aplicaciones AngularJS.
- [TestCafe](https://github.com/DevExpress/testcafe) - Pruebas de navegador automatizadas para la pila de desarrollo web moderna.
- [ava](https://github.com/avajs/ava) - 🚀 Runner de pruebas futurista de JavaScript
- [Cypress](https://www.cypress.io/) - Framework completo de prueba de extremo a extremo para cualquier cosa que se ejecute en un navegador y más allá.

### Assertion

- [chai](https://github.com/chaijs/chai) - Framework de aserción BDD/TDD para node.js y el navegador que se puede emparejar con cualquier framework de testing.
- [Enzyme](https://airbnb.io/enzyme/index.html) - Enzyme es una utilidad de testing de JavaScript para React que facilita afirmar, manipular y atravesar la salida de los componentes de React.
- [react testing library](https://github.com/kentcdodds/react-testing-library) - Utilidades de testing React DOM simples y completas que fomentan las buenas prácticas de testing.
- [Sinon.JS](https://github.com/sinonjs/sinon) - Test spies, stubs, y mocks para JavaScript.
- [expect.js](https://github.com/Automattic/expect.js) - Aserciones minimalistas de estilo BDD para Node.JS y el navegador.
- [proxyquire](https://github.com/thlorenz/proxyquire) - Stub nodejs's require.

### Coverage

- [istanbul](https://github.com/gotwarlost/istanbul) - Otra herramienta de coverage de código JS.
- [blanket](https://github.com/alex-seville/blanket) - Una biblioteca de coverage de código simple para JavaScript. Diseñado para ser fácil de instalar y usar, tanto para navegador como para nodejs.
- [JSCover](https://github.com/tntim96/JSCover) - JSCover es una herramienta que mide la cobertura de código para programas JavaScript.

### Runner

- [phantomjs](https://github.com/ariya/phantomjs) - Scriptable Headless WebKit.
- [slimerjs](https://github.com/laurentj/slimerjs) - Una herramienta similar a PhantomJS que ejecuta Gecko.
- [casperjs](https://github.com/casperjs/casperjs) - Utilidad de scripting y prueba de navegación para PhantomJS y SlimerJS.
- [zombie](https://github.com/assaf/zombie) - Pruebas de navegador increíblemente rápidas, full-stack y sin head usando node.js.
- [totoro](https://github.com/totorojs/totoro) - Una herramienta de prueba de navegador cruzado simple y estable.
- [karma](https://github.com/karma-runner/karma) - Espectacular Test Runner para JavaScript.
- [nightwatch](https://github.com/nightwatchjs/nightwatch) - Framework de prueba automatizado de interfaz de usuario basado en node.js y selenium webdriver.
- [intern](https://github.com/theintern/intern) - Una pila de prueba de código de próxima generación para JavaScript.
- [yolpo](http://www.yolpo.com) - Un intérprete de JavaScript declaración por declaración en el navegador.
- [puppeteer](https://github.com/GoogleChrome/puppeteer) - Headless Chrome Node.js API por el equipo oficial de Google Chrome.
- [webdriverio](https://github.com/webdriverio/webdriverio) - Framework de automatización de pruebas WebDriver de próxima generación para Node.js.
- [taiko](https://github.com/getgauge/taiko) - Una biblioteca Node.js con una API simple para automatizar navegadores basados en Chromium.
- [Playwright](https://github.com/microsoft/playwright) - Biblioteca Node.js para automatizar Chromium, Firefox y WebKit con una única API.
